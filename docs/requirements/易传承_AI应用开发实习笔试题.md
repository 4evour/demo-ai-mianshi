# AI 候选人线上初筛与项目真实性验证系统

## 任务

请为“AI 应用开发实习生”招聘场景设计一套 **AI 候选人线上初筛与项目真实性验证系统**，提交一份可执行的 **PRD + MVP 技术方案**。

系统需要覆盖以下主流程：

`岗位 JD → 简历上传与解析 → 岗位匹配分析 → 个性化问题生成 → 候选人线上文字作答 → AI 动态追问 → 评分与证据整理 → 人工审核`

AI 只提供辅助分析、评分建议和证据，最终是否进入下一轮由 HR 或面试官人工决定。

## MVP 边界

- 第一版只支持一个岗位，单批次约 100 位候选人。
- 输入包括岗位 JD、PDF 简历，以及可选的 GitHub、项目演示或作品链接。
- 面试形式仅考虑文字问答，暂不设计语音或视频面试。
- 暂不考虑面试排期、Offer 审批和复杂的多公司权限体系。
- 可以调用外部大模型，但必须明确敏感信息的处理边界。

## 交付内容

### 1. 产品与流程设计

说明系统的使用角色、完整业务流程、候选人状态流转、关键功能，以及人工审核介入的位置。

请区分 P0、P1 功能，并明确第一版不做什么。

### 2. 数据设计

设计主要数据对象及关键字段，至少包括：

- 岗位 JD
- 候选人及脱敏身份
- 简历原文件和结构化解析结果
- 项目经历及 GitHub、演示地址等作品链接
- 面试会话、问题、回答和追问记录
- 评分结果及对应证据
- 人工审核结果
- 模型调用和异常日志

不要求提供完整 SQL，但需要表达清楚各对象之间的关系。

### 3. AI 工作流

说明以下环节如何实现：

- 简历 PDF 的解析、字段结构化和错误校正
- 简历与岗位 JD 的匹配
- 根据候选人的经历和项目生成个性化问题
- 根据回答判断是否需要追问，以及如何生成追问
- 利用 GitHub、项目演示或其他材料辅助验证项目真实性
- 防止问题重复、跑偏或冒犯候选人
- 降低模型幻觉，并标记无法验证或证据不足的信息

请自行判断是否需要使用 RAG、Agent 或工作流编排，并说明选择理由。不要求为了使用某项技术而强行使用。

### 4. 评分与证据体系

设计一套可解释的评分方案，至少考虑：

- 岗位匹配度
- 专业技能掌握度
- 项目真实性
- 问题分析与解决能力
- 表达清晰度
- 风险点和待人工确认项

每项评分需要能够追溯到简历、作品链接或具体问答证据。请说明低置信度、证据不足和不同证据互相矛盾时如何处理。

### 5. 数据安全

说明姓名、手机号、邮箱、学校等敏感字段如何存储和脱敏，哪些信息可以发送给外部大模型，哪些不能发送，以及系统的访问权限、操作日志和数据保留策略。

### 6. 异常与人工兜底

至少覆盖以下情况：

- 简历解析失败或字段识别错误
- 候选人没有作品链接
- GitHub 仓库私有或暂时无法访问
- 候选人回答过短、答非所问或拒绝回答
- AI 生成了重复、不相关或不合适的问题
- 模型调用失败或超时
- AI 评分与面试官判断差异较大

### 7. MVP 技术方案

给出整体架构、主要技术选型和核心数据流，说明前端、后端、数据库、文件存储、大模型调用、检索、日志监控及人工审核模块之间的关系。

同时给出：

- 核心接口清单
- 关键性能或质量指标
- 1～2 周的 MVP 开发排期
- 上线前最重要的测试与验收项

### 8. 完整示例

提供一条完整示例。可以使用自己的脱敏简历，也可以虚构候选人，至少展示：

`简历解析结果 → JD 匹配结论 → 5 个个性化问题 → 某个回答触发的 2 个追问 → 最终评分报告及证据来源`

## 提交要求

- 格式：Markdown 或 PDF
- 建议篇幅：5～8 页
- 完成时间：收到题目后 48 小时内
- 建议投入：4～6 小时
- 不要求写代码或制作完整 UI，额外写代码不作为加分项
- 可以使用 AI 工具辅助，请在文档末尾简要说明 AI 用于哪些部分
- 提交后需进行约 15 分钟的方案讲解和问答

如果题目中存在未明确的信息，可以作出合理假设，但请在文档开头列明。

## 评分标准

| 评分项 | 占比 |
| --- | ---: |
| 业务理解与 MVP 取舍 | 20% |
| 产品流程与数据设计 | 20% |
| AI 工作流及技术可行性 | 25% |
| 评分体系与证据追溯 | 15% |
| 数据安全与异常处理 | 10% |
| 文档完整度与表达清晰度 | 10% |

# 个性化问题列表增删改设计

## 目标

让 HR 在 AI 生成个性化问题后，可以新增、删除并编辑问题，而不再局限于修改原有问题文本。候选人答题页只展示问题内容和必答状态，不暴露内部考察维度或考察目的。

## HR 端行为

- 每道未发布问题均可编辑问题内容、考察维度、考察目的和必答状态。
- 考察维度和考察目的允许为空，问题内容不能为空。
- 提供“新增问题”操作。新问题默认为空内容、空维度、空目的和必答。
- 每道未发布问题提供删除操作，不设置最低题数限制。
- 发布时至少需要一条问题，并拒绝问题内容为空的列表。
- 继续沿用现有前端草稿状态；刷新页面不保存未发布修改。

## 候选人端行为

- 当前题只展示问题内容及“必答”或“选答”状态。
- 保留主问题进度、追问进度和追问来源提示。
- 不展示考察维度和考察目的。

## 数据与接口

- 沿用现有 `Question` 结构，不新增服务端草稿接口。
- `dimension` 和 `purpose` 继续使用字符串字段，允许空字符串。
- 会话发布接口移除至少三道题的限制，改为至少一道题，并校验每道题的问题内容去除首尾空格后非空。
- 已发布会话及已有答题证据不允许通过本功能修改。

## 验证

- API 测试覆盖一道题可发布、空列表拒绝和空问题内容拒绝。
- UI 逻辑测试覆盖新增问题默认值和删除问题。
- 浏览器验证 HR 能新增、编辑、删除并发布问题。
- 浏览器验证候选人页面不显示考察维度或目的，只显示问题内容和必答状态。

## 非本次范围

- 未发布问题的服务端持久化。
- 拖拽排序。
- 修改已发布会话的问题。

# 多岗位多候选人审阅工作台设计

## 目标

将当前单页工作台拆分为可独立打开的岗位、候选人、问答、报告和对比页面。HR 在列表页保留全局上下文，点击岗位或候选人时在浏览器新标签页打开详情，避免简历、长回答和报告同时堆叠在一个页面。

## 页面结构

### `/` 岗位列表

- 展示全部岗位，而不是只加载第一条岗位。
- 每个岗位展示岗位名称、候选人数、已提交人数、已生成报告人数和创建时间。
- “新建岗位”在新标签页打开 `/jobs/new`。
- 点击岗位在新标签页打开 `/jobs/[jobId]`。
- 不在本页展示 JD、评分量表、简历、问答或完整报告。

### `/jobs/new` 新建岗位

- 保留现有岗位名称和 JD 解析流程。
- 创建成功后显示岗位详情页链接，由 HR 决定何时打开。

### `/jobs/[jobId]` 岗位详情

- 顶部展示岗位名称和精简要求摘要。
- 评分量表默认折叠，需要时展开查看。
- 保留 PDF 上传、OCR 纠错和简历解析能力。
- 展示该岗位的候选人列表，支持按状态查看并显示报告分数。
- 点击候选人在新标签页打开 `/candidates/[candidateId]`。
- 仅已生成报告的候选人可勾选对比；最多选择两人。
- 选择两人后在新标签页打开 `/jobs/[jobId]/compare?candidateIds=id1,id2`。

### `/candidates/[candidateId]` 候选人档案

- 展示姓名、状态、教育背景、技能、项目和荣誉。
- 保留生成问题、问题增删改、追问上限和发布答题链接能力。
- 提供独立入口，在新标签页打开问答记录和报告。
- 不直接渲染完整问答时间线或完整评分报告。

### `/candidates/[candidateId]/interview` 问答记录

- 展示面试状态、主问题进度、追问计数和人工复核状态。
- 每道题只显示题目摘要、类型、状态和简短回答预览。
- 完整回答默认折叠，HR 点击后展开；单个回答展开不会导致其他回答自动展开。
- 模型降级信息转换为简短可读提示，原始校验错误放入折叠技术详情。

### `/candidates/[candidateId]/report` 候选人报告

- 未提交时显示当前面试状态。
- 已提交但无报告时提供报告生成按钮，并在按钮附近显示加载和错误状态。
- 已生成报告时展示总分、维度、证据、优势、待确认和人工决策。
- 证据链接打开对应问答页面并定位到相关 turn。

### `/jobs/[jobId]/compare` 两人报告对比

- 必须提供两个不同候选人 ID。
- 两人必须属于当前岗位且均已生成报告，否则显示明确错误，不渲染失真的对比。
- 候选人为两列；对比总分、各岗位评分维度、证据置信度、优势、待确认和人工决策。
- 维度顺序使用岗位评分量表，不按模型返回顺序排列。
- 每列提供候选人报告页链接，并在新标签页打开。

## 新标签页规则

- 岗位列表到岗位详情、岗位详情到候选人详情、候选人详情到问答或报告、岗位详情到对比页，均使用普通链接的 `target="_blank"` 和 `rel="noreferrer"`。
- 列表页不因打开详情而丢失筛选和选择状态。
- 页面本身拥有稳定 URL，可刷新、收藏和直接访问。

## 数据与接口

- 继续使用现有 `StoreData`，不迁移或复制岗位、候选人、会话和报告。
- 抽取共享审核读模型，统一隐藏简历原文和脱敏模型输入，避免各页面重复拼装敏感数据。
- 岗位列表读模型增加候选人数、已提交人数和报告人数。
- 增加按 ID 获取岗位与候选人审核视图的接口；不存在或归属不匹配时返回 404。
- 对比页服务端校验两个候选人属于同一岗位且报告存在，不能仅依赖前端复选框限制。

## 页面状态与错误处理

- 每个页面独立处理加载、空数据、404 和接口错误。
- 岗位无候选人时显示上传入口，不展示空白列表。
- 报告生成失败时错误就地显示在报告按钮附近。
- 列表切换不触发外部模型调用；仅创建岗位、解析简历、生成问题、动态追问和生成报告调用 DeepSeek。

## 验证

- 数据测试覆盖岗位汇总计数和候选人审核视图的数据脱敏。
- 路由测试覆盖不存在的岗位/候选人、跨岗位对比、少于或多于两人、缺失报告。
- UI 测试覆盖最多选择两人、未生成报告不可选和新标签页链接属性。
- 浏览器验证岗位列表、岗位详情、候选人档案、折叠问答、报告和两人对比完整链路。
- 在 320、768、1024 和 1440 像素宽度检查无横向溢出；对比页在窄屏使用双列横向滚动，不压缩文字到不可读宽度。

## 非本次范围

- 跨岗位候选人对比。
- 三人及以上同时对比。
- 用户认证、多租户、数据库迁移和服务端分页。
- 修改已提交问答或自动改变人工决策。
